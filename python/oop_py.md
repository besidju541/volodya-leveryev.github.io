# Объектно-ориентированное программирование в Python  

---

## Введение

Объектно-ориентированное программирование (ООП) — одна из фундаментальных парадигм программирования, наряду с процедурной и функциональной. Основная идея ООП заключается в представлении программной системы в виде совокупности взаимодействующих объектов, каждый из которых отражает сущность предметной области.

Python является полностью объектно-ориентированным языком: числа, строки, функции, классы и модули — всё является объектами. Это делает Python удобным инструментом для изучения и практического применения ООП.

---

## 1. Парадигмы программирования

### 1.1 Процедурное программирование
Программа представляется в виде последовательности инструкций и функций.

### 1.2 Функциональное программирование
Основной акцент делается на использовании чистых функций и неизменяемых данных.

### 1.3 Объектно-ориентированное программирование
Программа строится как система объектов, взаимодействующих друг с другом через методы.

---

## 2. Основные понятия ООП

### Класс
Класс — это абстрактный шаблон, описывающий свойства и поведение объектов.

### Объект
Объект — конкретный экземпляр класса, размещённый в памяти.

### Атрибут
Атрибут — переменная, характеризующая состояние объекта.

### Метод
Метод — функция, определённая внутри класса и предназначенная для работы с объектами.

---

## 3. Классы и объекты в Python

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
```

Создание экземпляра:

```python
p = Person("Ivan", 20)
```

---

## 4. Конструкторы и деструкторы

Метод `__init__` используется для инициализации объекта.  
Метод `__del__` вызывается при удалении объекта сборщиком мусора.

---

## 5. Атрибуты класса и объекта

```python
class Student:
    university = "State University"

    def __init__(self, name):
        self.name = name
```

Атрибут `university` является общим для всех экземпляров класса.

---

## 6. Инкапсуляция

Инкапсуляция обеспечивает контроль доступа к данным объекта.

```python
class Account:
    def __init__(self, balance):
        self.__balance = balance
```

---

## 7. Свойства (property)

```python
class User:
    def __init__(self, age):
        self._age = age

    @property
    def age(self):
        return self._age

    @age.setter
    def age(self, value):
        if value < 0:
            raise ValueError("Возраст не может быть отрицательным")
        self._age = value
```

---

## 8. Наследование

```python
class Student(Person):
    def __init__(self, name, age, group):
        super().__init__(name, age)
        self.group = group
```

Наследование способствует повторному использованию кода.

---

## 9. Полиморфизм

```python
class Dog:
    def speak(self):
        print("Woof")

class Cat:
    def speak(self):
        print("Meow")
```

Полиморфизм позволяет работать с объектами разных классов через единый интерфейс.

---

## 10. Абстракция

```python
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass
```

Абстрактные классы определяют интерфейс для дочерних классов.

---

## 11. Композиция и агрегация

Композиция — жёсткая форма связи объектов.  
Агрегация — слабая форма связи.

---

## 12. Магические методы

Магические методы позволяют переопределять стандартное поведение объектов.

Примеры:
- `__str__`
- `__repr__`
- `__eq__`
- `__len__`

---

## 13. Статические и классовые методы

```python
class Utils:
    @staticmethod
    def add(a, b):
        return a + b

    @classmethod
    def info(cls):
        return cls.__name__
```

---

## 14. Преимущества и недостатки ООП

### Преимущества
- структурированность кода;
- повторное использование;
- удобство сопровождения;
- масштабируемость.

### Недостатки
- сложность проектирования;
- избыточность для небольших программ.
